# 前端性能优化

1. 代码优化: 压缩与合并, 懒加载, 按需加载

2. 页面渲染优化: 全局封装基础请求方法, 骨架屏, CDN加速

3. DOM操作优化: 减少直接操作dom, 如果非要操作, 几次dom合并在document.createDocumentFragment()中操作, 再统一插入到dom中

4. 用户交互优化: 平滑过渡(使用动画和过渡效果,提高用户体验), 表单验证(实时表单验证, 减少用户提交错误数据的可能性)


# 网络层面的优化

1. 数据传输优化: 对传输的数据进行压缩,减少带宽;使用HTTP2/HTTP3等协议, 支持多路复用和头部压缩;对于实时通信的场景可以用websocket或者sse来替代轮训

2. CDN与计算边缘: 使用CDN分发静态资源,减少延迟;将计算任务推到离用户最近的位置, 减少数据传输距离.


# 大型中后台后台使用微服务前端使用微前端

1. 微服务: 将应用拆分成多个模块, 每个模块是一个独立的服务, 服务之间通过网关进行通信, 一个微服务通常对应相应的微前端子项目

2. 异步处理: 对于不影响用户体验, 耗时间较长的操作在右上角建立异步任务中心, 显示异步任务的名称,进度,结果, 例如excel大数据量导入数据库等


# 监控与日志

1. 性能监控: 利用应用的日志收集发现性能瓶颈, 给出解决方案;例如当接口请求操作3秒时, 前端会把该接口名称,以及当时对应的浏览器信息, 网速

2. 错误报告: 捕获全局的js错误,当发生错误后请求错误上传接口.定期查看错误日志系统修复错误

# 安全性与稳定性

1. 后端权限控制, 前端对提交的数据(富文本输入框数据)进行xss处理

2. 回滚脚本: 当发生异常时, 如果影响用户使用, 一键执行回滚脚本回滚整个应用
